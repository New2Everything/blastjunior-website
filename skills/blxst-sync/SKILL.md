# BLXST Sync Skill

> 用于同步项目文档，确保知识库与实际状态一致

## 触发方式

- 输入 `/blxst-sync` 或 `/sync`
- 手动触发（老K在睡前/下线前执行）

---

## 执行流程

### 步骤1：列出需要检查的项目

**必须检查以下文件的状态：**

| 文件 | 检查内容 |
|------|----------|
| `BLXST-knowledge-graph.md` | API/数据库/页面是否有变化？ |
| `BLXST-rules.md` | 是否有新增规则？ |
| `BLXST-status.md` | 进度是否需要更新？ |
| `knowledge/hado-business.md` | 是否有新的HADO知识？ |

### 步骤2：询问老K确认

**列出发现的差异，逐条确认：**

```
🔍 发现以下需要同步的内容：

1. [文件] - [差异描述]
2. [文件] - [差异描述]
...

请逐条确认是否同步，回复格式：
- "1,2,3" = 同步 1,2,3
- "1" = 只同步 1
- "全部" = 全部同步
- "跳过" = 暂不同步
```

### 步骤3：执行同步

确认后，逐条更新文件内容。

### 步骤4：完成汇报

```
✅ Sync 完成！

已同步：
- [文件1]: [更新内容]
- [文件2]: [更新内容]

未同步：
- [原因]
```

## 自动检测规则

### 知识图谱检查项

- [ ] 新增/删除的 API 端点
- [ ] 新增/删除的数据库
- [ ] 新增/删除的前端页面
- [ ] API 端点返回数据变化

### 规则检查项

- [ ] CORS 配置变化
- [ ] 数据源绑定变化
- [ ] 新的禁止事项
- [ ] 架构原则变化

### 进度检查项

- [ ] 新完成的功能
- [ ] 待处理的任务
- [ ] 遇到的问题

### HADO知识检查项

- [ ] 是否有新的业务规则
- [ ] 是否有新的赛事信息
- [ ] 是否有新的资源链接

---

## 注意事项

1. **必须逐条确认** - 不可以直接静默同步
2. **对比实际状态** - 用 curl/API 调用验证当前状态
3. **记录更新日志** - 每次同步后追加更新日志
4. **保持简洁** - 只记录变更，不重写整个文件
