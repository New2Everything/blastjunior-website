# BLXST Registry Update Proposal Policy v0

Purpose: create dry-run proposals for registry / routing-rule updates.

This is not an auto-repair tool.

## Core Law

This stage creates a proposal only.

It must not write registry files, website source, D1, R2, KV, Workers, Cloudflare, git, or deployment state.

## Non-Hardcoded Principle

Proposal fields are non-exhaustive.

Future resource types must safe-stop into review instead of being ignored.
