# BLXST Runtime Origin Policy v0

Purpose: define authorized Learning V2 runtime origins without hardcoding them into scripts.

## Core Law

Runtime origins are policy-driven.

Unknown origins safe-stop before mutation.

User-direct BLXST tasks should use `/blxst`.

Scheduled, autonomous, and controlled phases do not require `/blxst`, but they must carry an authorized origin.

This policy does not enable production mutation or production deploy.
