# BLXST Autonomous E2E Orchestrator Policy v0

Purpose: run the autonomous dry-run chain from task text to final safe-stop/readiness report.

This is not real apply, not auto-repair, and not deploy.

## Core Law

The orchestrator follows report outputs and policy decisions. It does not hardcode future resources.

It must not mutate registry, website source, D1, R2, KV, Workers, Cloudflare, git, or deployment state.

A ready result only means readiness for a later controlled context.
