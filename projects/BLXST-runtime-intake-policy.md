# BLXST Runtime Intake Policy v0

Purpose: create provenance intake envelopes for user/OpenClaw BLXST content before any mutation.

## Core Law

Runtime intake is envelope-only.

It must not write registry, D1, R2, KV, Workers, website source, Cloudflare, git, or deployment state.

## Provenance Law

User-submitted content can be treated as submitted evidence, but not automatically published.

AI-extracted structure is structure only, not factual authority.

Unknown content safe-stops into review.

Content family hints are non-exhaustive and policy-driven.
