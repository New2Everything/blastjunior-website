# 项目执行规范 (RULES.md)

所有项目的执行规范。

## ⚡ 核心原则

### 1. 精简原则
- 项目文件不超过 100 行，只记录关键信息
- 不在项目文件里放完整代码或长文本

### 2. 读取原则
- 项目文件每 session 只读取一次（开头读取后记住）
- 后续需要细节时再针对性读取

### 3. 浏览器原则
- 每次用 Agent Browser 前，先列出要获取的 3-5 个点
- 带着目标去，一次性获取足够信息再分析

### 4. 迭代原则
- 每次只设定一个小目标
- 不追求一次做完，先快速迭代

### 5. 存档原则
- 每次会话结束前，更新项目文件的进度/待办
- 关键决策必须写下来
