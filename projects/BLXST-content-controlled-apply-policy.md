# BLXST Content Controlled Apply Policy v0

Purpose: define the controlled context and disabled executor skeleton for future content writes.

## Core Law

Real content apply is disabled by default.

A ready content rehearsal is not permission to write.

A controlled content apply context must be explicit and machine-readable.

This phase installs the context gate and disabled executor skeleton only.

It must not write D1, R2, KV, Workers, website source, registry, Cloudflare, git, or deployment state.
