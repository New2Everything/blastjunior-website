## 一、用户研究

### 1.1 用户群体分析

| 用户类型 | 描述 | 占比预估 | 核心需求 |
|----------|------|----------|----------|
| **球员** | 兰星少年俱乐部注册选手 | ~50人 | 个人档案、比赛记录、战绩统计、所属战队信息 |
| **家长** | 选手的家长们 | ~100人 | 孩子比赛情况、赛季安排、成长轨迹、俱乐部通知 |
| **粉丝** | HADO爱好者、观众 | 大量 | 赛事资讯、战队/选手信息、精彩瞬间、比赛直播 |
| **赞助商** | 潜在/现有赞助商 | 少量 | 俱乐部实力、赞助方案、联系方式、曝光效果 |

### 1.2 用户画像（Persona）

#### 🎮 球员 - 小明
- **年龄**：12-16岁
- **使用场景**：
  - 训练之余刷网站看自己和队友的表现
  - 赛前查看对手信息
  - 赛后回顾个人数据
- **痛点**：想看详细数据但找不到入口
- **期望**：个人主页、得分统计、比赛记录

#### 👨‍👩‍👧 家长 - 小明妈妈
- **年龄**：35-45岁
- **使用场景**：
  - 周末查看孩子比赛结果
  - 了解赛季赛程安排
  - 关注孩子成长轨迹
- **痛点**：信息分散，找不到孩子的比赛记录
- **期望**：一键查看孩子所有比赛、简洁易懂

#### 📱 粉丝 - 小红
- **年龄**：18-30岁
- **使用场景**：
  - 休闲时浏览HADO资讯
  - 关注喜欢的战队/选手
  - 观看比赛精彩瞬间
- **痛点**：内容不够丰富，没有互动
- **期望**：最新资讯、精彩图库、社区互动

#### 💼 赞助商 - 王总
- **年龄**：30-50岁
- **使用场景**：
  - 评估合作价值
  - 了解俱乐部战绩
  - 查看曝光方案
- **痛点**：找不到专业的赞助介绍页面
- **期望**：专业、量化数据、联系方式清晰

## 七、风险与约束

### 7.1 数据真实性约束

> 参考 BLXST-rules.md 核心原则

- ✅ 允许：AI生成的新闻（需标注）
- ❌ 禁止：虚构的战队/选手/比赛结果
- 原则：所有数据必须来自 D1/R2 真实数据库

### 7.2 技术约束

- 单API架构原则：所有端点走 blast-homepage-api
- CORS配置：仅允许 blastjunior.com
- Gallery数据必须从R2读取，禁止伪造

#### CORS 安全配置

所有 API 响应必须包含：
```
Access-Control-Allow-Origin: https://blastjunior.com
```

**禁止：**
- ❌ 通配符 `*`
- ❌ 空格分隔多域名：`"https://blastjunior.com https://www.blastjunior.com"`

#### 数据完整性（必须检测）

```
检测模式（grep）：
- "暂无数据"         → 失败
- "加载失败"         → 失败
- "加载错误"         → 失败
```

注意：单个字段为"-"是允许的（如选手没有战队），只有"暂无数据"/"加载失败"才是真正的数据缺失。

#### JS错误（必须检测）

```
agent-browser errors：
- 包含 "error"     → 警告
- 包含 "failed"    → 警告
- 包含 "exception"  → 警告
```

#### 未定义数据（必须检测）

```
grep snapshot：
- "undefined"       → 警告
- "null"           → 警告
- "NaN"            → 警告
```
