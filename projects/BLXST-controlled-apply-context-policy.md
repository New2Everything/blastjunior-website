# BLXST Controlled Apply Context Policy v0

Purpose: define the only controlled context under which a future registry apply executor may write registry/routing/policy files.

## Core Law

Real apply is disabled by default.

A ready dry-run report is not permission to write.

A controlled apply context must be explicit and machine-readable.

## This Phase

This phase installs the controlled apply context gate and disabled executor skeleton only.

It must not write registry files, website source, D1, R2, KV, Workers, Cloudflare, git, or deployment state.
